Description

Let's blend everything you've done in the previous stages into a user-friendly game!

Add a menu to the game with Place, Hint, New Puzzle, and Exit options.

Pass the file name for the crossword scrabble as a command-line argument — for example, crosswordScrabble.txt

Objectives
  • Read the JSON file that contains a list of puzzle objects with an example as shown below:
{"letter": "LFALYPU",

"words":[

   {"word":"Playful","position":"6,5","direction":"Acr","hint":"fond of games and amusement"},

   {"word":"fall","position":"2,3","direction":"Acr","hint":"Move from a higher to a lower level, typically rapidly and without control"},...

]

}
  • At the start, get a random object and fill the crossword grid cell with blank space where user input from the user is expected. On the New Puzzle command, read another puzzle randomly and display a space where users need to fill it.
  • The Place command fills in a word in the crossword, for example — Place fall 2 3 ACR. If the word is longer than the available space, print Incorrect word. The word is too long. If the word is shorter, print The word is shorter than expected. If the word is not correct, print Wrong word, please try again. and display the hint on the following line;
  • The hint command describes the expected word. It takes the row and column position of the word as the starting position and the direction — hint 2 3 ACR;
  • For direction and/or position parameters — if the position is out of range or the direction is invalid, print Invalid position or direction.
  • When the puzzle is solved, print Congratulations! Would you like to load another puzzle? Yes/No. If Yes, load another puzzle. If No, terminate the application.

The greater-than symbol followed by a space (> ) represents the user input. Note that it's not part of the input.
